Промпты для нейросетей: как писать эффективные запросы

5 мин чтения · 6 июля 2026 · промпты AI

Промпт (prompt) — это запрос, который вы отправляете нейросети. Качество ответа напрямую зависит от того, как составлен промпт. Хороший промпт — это чёткая инструкция, которая не оставляет пространства для двусмысленностей.

Универсальная формула промпта

Для любой нейросети работает одна и та же структура:

[Роль] + [Контекст] + [Задача] + [Формат] + [Ограничения]

Пример плохого промпта: «Напиши пост для Instagram»

Пример хорошего промпта: «Ты — SMM-менеджер (роль). Мы запускаем AI-сервис для автоматизации (контекст). Напиши пост-знакомство на 500 знаков (задача). Используй дружеский тон, добавь 3 хештега (формат). Без эмодзи и восклицательных знаков (ограничения)»

Техники промптов для ChatGPT

1. Chain-of-Thought (Цепочка рассуждений)

Попросите AI рассуждать шаг за шагом. Вместо «Реши задачу» — «Реши задачу по шагам, объясняя каждый шаг». Это dramatically улучшает точность для сложных задач.

2. Few-Shot (Примеры в промпте)

Дайте 2-3 примера того, что хотите получить. ChatGPT отлично улавливает паттерны:

«Напиши 3 варианта приветствия для email-рассылки. Формат как в примере:
Пример: "Привет, {имя}! 👋 Готовы к переменам?" → "Привет, {имя}! 🚀 Ваш чек-лист готов."»

3. Ролевой промптинг (System Prompt)

Чётко задайте роль. ChatGPT работает лучше, когда знает контекст:

  • «Ты — эксперт по маркетингу с 10-летним опытом»
  • «Ты — Python-разработчик уровня Senior»
  • «Ты — копирайтер, специализирующийся на продающих текстах»

Промпты для Midjourney

Для Midjourney важны визуальные описания и параметры. Базовая структура:

[Объект] + [Детали] + [Окружение] + [Освещение] + [Стиль] + [Параметры]

Пример: Viking longship emerging from thick fog, photorealistic, dramatic lighting, cinematic, 8k --ar 16:9 --v 6

Ключевые параметры Midjourney:

  • --ar 16:9 — соотношение сторон (широкий формат)
  • --v 6 — версия модели (сейчас актуальна v6)
  • --s 250 — стилизация (50-1000, чем выше — тем артистичнее)
  • --iw 2 — вес изображения (при работе с reference)

Промпты для Claude

Claude отлично работает с кодом и техническими задачами. Лучшие практики:

  • Один запрос — одна задача: Claude лучше фокусируется на конкретной проблеме
  • Прикрепляйте контекст: Дайте код, документацию или описание архитектуры
  • Используйте XML-теги: Claude понимает структурированные запросы

«Review this PR. Check: logic errors, edge cases, performance issues, security vulnerabilities, code style. Here's the diff: ...»

Промпты для Stable Diffusion

SD требует более технических описаний. Важно указывать:

  • Negative prompt: что НЕ должно быть на картинке
  • Sampler и Steps: DPM++ 2M Karras, 20-30 steps
  • CFG Scale: 7-9 для баланса следования промпту

Пример: Majestic mountain peak at golden hour, clouds swirling around summit, alpine lake reflection, photorealistic С negative prompt: blurry, low quality, deformed, watermark

Чек-лист хорошего промпта

  1. ✅ Определена роль AI
  2. ✅ Дан контекст / background
  3. ✅ Чётко сформулирована задача
  4. ✅ Указан формат ответа
  5. ✅ Добавлены ограничения (длина, тон, стиль)
  6. ✅ Использованы примеры (few-shot) для сложных задач
  7. ✅ Запрос на русском или английском (в зависимости от модели)
